Each row is the squash commit on `main` (visible in `git log --first-parent`). Sub-phase commits existed during mid-cycle coordination and are no longer in git history; per-phase decisions live in `docs/decisions.md`.

## Per-phase highlights

### Phase 3: Talent Scout port

Talent Scout was originally shipped as a standalone Lovable app and lifted into HQ as an admin-only sub-route. The port covered schema augmentation, roles CRUD + wizard, the chunked Gmail pull pipeline (`BATCH_SIZE=8`), CandidateDetail + status dropdown + re-eval, the Final Review packet (pdf-lib + signed-URL email), referral ingestion (forward-chain walker), pg_cron + 6 cron jobs (scheduled pulls + 3 watchdogs + storage cleanup + monthly spend reset), scorecard refinement, and substantive describer fields. Per-sub-phase decisions in `docs/decisions.md` Phase 3 sections.

### Phase 4: Venue Scout port

Full 1:1 port from `mirror-nyc-venue-scout-pro` shipped to production 2026-05-13. Producer flow: New Scout → Brief upload (DOCX + AI parse) → Sheet Prompt → Sheet Upload (XLSX/CSV + AI enrichment) OR direct AI venue research → Sourcing Report → Shortlist (matrix + photo upload) → Review → Compiling (per-venue overview generation) → Deck Prep → Generating (Google Slides deck) → deck opens in new tab. AI surface uses `pause_turn` continuation, `writeFailure` CAS guards on all 3 AI edge functions, 180-360s `WORK_TIMEOUT_MS`, URL-fallback helpers. Phase 4 Revision rebuilt the Brief as a 3-step stepper (Event → Venue → Review), added venue-side fields the AI sourcing prompt needs, added `vs-generate-brief-overview`, made the Revisit nav always-visible. Per-sub-phase decisions in `docs/decisions.md` § "Phase 4 cutover + port plan locked decisions" + § "Phase 4 Revision".

### Phase 5.1 to 5.6: HQ Core foundations to interaction primitives

The first six HQ Core phases established the data backbone (5.1 permission tiers + `notes_log` + AppShell; 5.2 entities + clients-vendors split + DataTable / BoardView / TimelineView / CalendarMonthView primitives; 5.3 Calendar + Outlook + `promote_outlook_to_project` RPC) and the cross-tool surfaces (5.4 Wiki + Settings + admin pre-provisioning; 5.5 Notifications + Activity Feed + Search + Sign-in; 5.6 cross-cutting interaction primitives + inline-edit + global default views via `users.is_owner` + `saved_views.scope`).

### Phase 5.7: HQ Core UI overhaul

Largest single HQ Core shipping arc. Quick-Add wiring; @-mentions feed (note_mentions) + append-only Notes on Tasks and Deliverables; fixed-position StickySaveBar; deliverables status reshape (drop In Progress; auto-task lifecycle helper); global FilterBar distinct-values picker; project_members general-team join; Calendar overhaul (Day/Week/Month + My Tasks); Settings Lookup Lists merge + Wiki image upload pipeline (private bucket, 1-year signed URLs, diff-on-save cleanup); Vendor Files & Assets URL list; Profile + Settings split; per-user `vendor_ratings`. Sets the HQ Core UX baseline that subsequent phases iterate against.

### Phase 5.8: HQ v1 release + security audit + cleanup

Tier 1/2/3 audit pass closing the v1 release window. Security migrations covered function search_path pinning, REVOKE EXECUTE on SECURITY DEFINER fns, RLS init-plan rewrite, storage bucket listing, and `credentials` column encryption via pgsodium AEAD. Surface reduction closed HIGH XSS, wrapped wiki HTML in DOMPurify, dropped 28 unused shadcn UI components + 23 unused deps. Auth-pre-provision hotfix restored the Phase 5.4 swap block + flipped all 42 FKs pointing at `public.users.id` to ON UPDATE CASCADE.

### Phase 5.9: Bulk import

CSV-only bulk import across Projects + Vendors + Venues. Cross-cutting primitive shape: shared `BulkImportPage` orchestrator + `UploadStep` + `MapStep` + `DedupeStep` + virtualized `ImportGrid`; per-entity `bulk_import_commit_*` SECURITY DEFINER RPCs; map step for unknown refs; admin-only; dedupe defaults to skip with per-row override; configurable column picker; fresh HQ-side parser (`_shared/hqSheetParser.ts`); audit page at `/settings/bulk-import/history`; bulk-import undo (7-day window). Event Day Rate import + `vendors.nationwide` + `general_email` columns landed during the cycle.

### Phase 5.10: venues.about_venue rename + AI About Venue generator

`venues.notes` renamed to `venues.about_venue` (OID-preserving) because `notes` collided with the polymorphic `notes_log` table. New `hq-generate-venue-about` edge function is the first `callClaude('hq', ...)` consumer; tool-less by design (evergreen `ABOUT_VENUE_SYSTEM` prompt + `web_search` + `tool_choice: auto`). Generate / Regenerate buttons on VenueDetail + VenueEdit. VenueEdit + VenueDetail layout refresh with 2-column label-above Details cards, `card-headbar` section headers, Master Venue Deck Slide card, Features as tag input. First responsive `@media` breakpoints. Plus v1 codebase triage: 66 routes converted to `React.lazy` behind Suspense, 23 unused deps dropped, initial bundle 2.24 MB → 586 KB.

### Phase 5.11: UX and structural consistency review

Design-system audit sequence. Focus rings (shared `:focus-visible` coral ring across `.btn` family + view switcher / tabs / toggle / checkbox / filter chips / saved-views / rail nav / text links / combobox picker / topbar controls). Mobile shell: rail collapses to off-canvas hamburger drawer below 1024px; primary + secondary tap targets ≥44px on mobile. Tokens: `--info` set to canonical `#06B6D4`, new `--purple` `#B57BF5`; 16 hardcoded cyan/purple hex literals replaced. Coral link contrast: in-content links switched from `--primary` to `--primary-hover`. `hq-` chrome convergence: dead Phase 5.2.1 `hq-` block removed; Home + ClickPillCell migrated to unprefixed canonical classes. Detail-page polish + list-table standardization. Managed Project Tags + Venue Features lookups. Structural component extraction (`DField`, `HQFormField`, `ContactsCard`, `WebsiteActionButton`, `prettyHost`, `ListPageChrome`).

### Phase 5.12: Venue Scout full audit + UX update + cleanup

Largest single VS shipping arc (28 sub-phases). Schema reshape: `vs_candidate_venues.source` widened to include `'hq_pool'`; `dedupe_meta` jsonb shape; `brief_data` shape lock (5 keys flip from `string` to `string[]`); neighborhoods nested in cities; venue_types DB-driven with case-insensitive uniqueness; city_aliases lookup. Edge function patterns: two new advisory-lock kickoff RPCs (`vs_research_try_acquire_kickoff`, `vs_deck_try_acquire_kickoff`); CAS guards on final UPDATEs (`current_step` + `pipeline_error IS NULL`); fit-rescue pattern for HQ pool venues; `vs-research-single-venue` three-gate auth pattern; `vs-regenerate-venue-overview` synchronous single-venue Pass 2; `vs-delete-scout` per-bucket storage cleanup. Prompt audit: `key_features` as evergreen tags; ABOUT_VENUE_SYSTEM cap widen to 100-150 words; Retail bias soften; Event Overview voice tuning; ParsedPreview selective-apply checkboxes. UX: `.tbl` matrix decouple, back-crumb relocation to TopBar, Lookup Lists shared component, ScoutPageHeader 3-zone primitive, VSPageField primitive, ScoutPhaseBreadcrumb single-row, DateRangePicker, VenueMatrixRow shared 7-column layout, Final Review + Deck Prep consolidation, Loading shell honest server-driven progress. 10 migrations + 9 edge functions touched cumulatively across the cycle. Phase 5.12.8 (Brief client-logo web search) was CUT.

7 sub-phases collapsed into one squash. Nav context parity with Venue Scout: `BrandTS` ("Talent Scout" with coral "Scout"), `TS_TOOL_ITEMS`, cross-tool links, `IconTalentScout` (two-person) + `IconVenueScout` (two-buildings) replace retired `IconScout`, search hidden on `/talent-scout/*`, back-crumb label simplified to "Back" HQ-wide. Pipeline smoke: pull + re-eval + final review packet all clear.
